Operators are categorized into several types: *arithmetic*, *comparison*, and *Relational, Logical assignment, and In* operators 


# Arithmetic Operations: Financial & Operational Calculations

cost_of_goods = 250000 - 180000       # Difference: Gross profit → 70,000
gross_margin = 1500 * 45.99           # Product: Units sold × price

leftover_inventory = 1007 % 50

# Relational Operators: Performance & Threshold Checks

Basic Knowledge II: Data Collection

A. List

A list is an ordered collection of items which can be of `varied data types`. Lists are mutable, meaning they can be altered after their creation.

1.1 Creating a List

They are defined by square brackets [ ], with items separated by commas 


# Sales data: List of quarterly revenues (in thousands)
sales = [250, 300, 280, 320]  # Q1, Q2, Q3, Q4 → all integers

# Mixed-type list: Customer record (ID, name, region, contact)
customer_record = [1001, 'Linda Chen', 'Taiwan', 0935123456]  

# Empty list: Initialize for dynamic data collection (e.g., new leads)
new_leads = []

1.2 Accessing List Elements

List items can be accessed by their index . Each item has an index indicating its position in the list. The first item in a list is at index 0, the second at index 1, and so on 


# Product list: Top-selling items in e-commerce store
products = ["laptop", "smartphone", "tablet"]
print(products)  # Outputs: ['laptop', 'smartphone', 'tablet']

# Access second product (index 1)
print(products[1])  # Outputs: 'smartphone'

# Access last item using negative indexing
print(products[-1])  # Outputs: 'tablet'

1.3 Modifying Lists


# Update product due to inventory change
products[1] = "wireless earbuds"  # Replace smartphone with trending item

# Add new product to end of list
products.append("smartwatch")

# Remove discontinued item
products.remove("laptop")

print(products)  # Updated product catalog

# Insert new product at position 1
products.insert(1, "fitness tracker")

# Sort products alphabetically for reporting
products.sort()
# products.sort(reverse=True)  # For descending order

1.4 List Operations

  • concatenation, repetition, and **methods** like sort(), reverse(), and extend() 


# Another category: Physical store inventory
accessories = ["charger", "case", "screen protector"]

# Combine online and in-store product lists
full_inventory = products + accessories
print(full_inventory)

# Repeat seasonal sales targets for multiple regions
base_targets = [100, 120, 90]
expanded_targets = base_targets * 2  # Apply same targets to two regions
print(expanded_targets)  # [100, 120, 90, 100, 120, 90]

# Check if a product is in inventory
"tablet" in products  # Returns True or False 

# Find total number of products
len(full_inventory)  # Useful for dashboard KPIs

# Loop through sales team members for performance review
sales_team = ['Alice Lin', 'David Wang', 'Sofia Huang']

for member in sales_team:
    print(f"Reviewing performance: {member}")

1.5 Slicing a Lis

Keep in mind that Python stops one item **before the second index** (but not including) you specify. 


# Slicing: Analyze top performers from leader board
leaderboard = ['Charles', 'Martina', 'Michael', 'Florence', 'Eli']

print(leaderboard[0:3])   # Top 3 performers
print(leaderboard[1:4])   # Middle-ranked staff

print(leaderboard[2:])    # From 3rd onward
print(leaderboard[-3:])  # Last 3 employees → potential at-risk performers

B. Tuple

1.1 Creating a Tuple

Similar to a list in terms of indexing, nested objects, and iteration. However, unlike lists, tuples are immutable. Once a tuple is created, it cannot be altered. Tuples are defined by parentheses (), with items separated by commas 


# Immutable data: Fixed dimensions (e.g., store size in sq ft)
store_size = (1200, 800)  # Width, Length
print(store_size)

1.2 Accessing Tuple Elements


print(store_size[0])  # Access width → 1200

# store_size[0] = 1500  # ❌ Error! Tuples are immutable → protects critical data

# Combine dimensions from multiple stores
store_b_size = (900, 700)
all_store_sizes = store_size + store_b_size
print(all_store_sizes)  # (1200, 800, 900, 700)

C. Dictionaries

A dictionary in Python is an unordered collection of data values, used to store data values like a map. The dictionaries hold key-value pairs. Keys in a dictionary must be unique and immutable, which typically are strings, numbers, or tuples.

1.1 Creating a Dictionary

Dictionaries are defined by (curly) braces { } with each item being a pair in the form key: value. 


# Dictionary: Customer profile (key-value pairs)
customer = {
    "name": "Alice Wu",
    "age": 25,
    "city": "Tainan"
}
print(customer)

1.2 Accessing Dictionary Elements

we can access the values in the dictionary by referring to its key 


print(customer["name"])  # Retrieve customer name

1.3 Modifying Dictionaries

Dictionaries are mutable. we can add new key-value pairs, modify values, or delete key-value pairs 


# Update customer info
customer["age"] = 30
customer["profession"] = "Marketing Manager"  # Add new attribute

# Remove outdated field
del customer["city"]
print(customer)

1.4 Dictionary Methods

Dictionaries have built-in methods, such as `get()`, `keys()`, `values()`, and `items()` 


# Extract keys and values for reporting
print(customer.keys())    # ['name', 'age', 'profession']
print(customer.values())  # ['Alice Wu', 30, 'Marketing Manager']

# Loop through customer survey respondents
survey_responses = {
    'jen': 'python',
    'sarah': 'c',
    'edward': 'rust',
    'phil': 'python',
}

for respondent in survey_responses.keys():
    print(f"Respondent: {respondent.title()}")

D. Sets

1.1 Creating a Set

A set is an unordered collection of unique items. Sets are are ideal for performing mathematical set operations like unions, intersections, and differences.
Sets are defined by curly braces {} or the set() function, and they automatically remove duplicate items 


# Set: Unique categories (no duplicates) – e.g., customer feedback tags
feedback_tags = {"refund", "delivery", "quality", "refund", "support", "delivery"}
print(feedback_tags)  # Outputs unique tags only

1.2 Accessing Set Elements

we can loop through the set items, or check if a value is present. 


if "delivery" in feedback_tags:
    print("Delivery issue reported → escalate to logistics team")

1.3 Modifying Sets

We can add items to sets using the add() method, and multiple items using the update() method. Items can be removed using remove() or discard() 


# Add new category
feedback_tags.add("pricing")

# Remove resolved category
feedback_tags.discard("support")
print(feedback_tags)

1.4 Set Operations

Sets are ideal for mathematical operations like unions (|), intersections (&), and differences (-) 


# Set operations for market analysis
digital_channels = {"email", "social", "search"}
traditional_channels = {"TV", "radio", "print", "social"}

all_channels = digital_channels | traditional_channels       # Union
shared_channels = digital_channels & traditional_channels   # Intersection
digital_only = digital_channels - traditional_channels       # Difference
